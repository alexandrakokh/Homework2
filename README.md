# Интернет-магазин

Проект интернет-магазина, который будет развиваться на протяжении курса. Текущая версия реализует базовую модель товаров, категорий и строгую проверку типов при операциях.

## Функционал текущей версии

В рамках текущего задания реализованы следующие возможности:

- **Иерархия товаров**: базовый класс `Product` и два класса-наследника:
  - `Smartphone` — расширен атрибутами: производительность (`efficiency`), модель (`model`), объём памяти (`memory`), цвет (`color`).
  - `LawnGrass` — расширен атрибутами: страна-производитель (`country`), срок прорастания (`germination_period`), цвет (`color`).
- **Строгая типизация при сложении**: сложение товаров (`__add__`) разрешено только для объектов одного и того же класса. Попытка сложить, например, смартфон и газонную траву вызывает `TypeError`. Проверка реализована через `type(self) is not type(other)`.
- **Защита категорий**: метод `Category.add_product` принимает только экземпляры `Product` или его наследников. Для проверки используется `isinstance(product, Product)`. Любая попытка добавить посторонний объект (строку, число и т.п.) приводит к ошибке.
- **Статистика и вывод**: категории умеют считать общее количество товаров и формировать читаемое строковое представление списка товаров.
- Базовый абстрактный класс `BaseProduct` с обязательными методами `get_total_cost()` и `get_description_preview()`.
- Классы `Product`, `Smartphone`, `LawnGrass` с корректной цепочкой наследования.
- Миксин `LoggingInitMixin`: при создании объекта выводит в консоль информацию о классе и параметрах.
- Класс `Category`: группировка товаров, подсчёт общей стоимости категории.
- Логика сложения товаров одного типа (`__add__`).
- Набор тестов на `pytest` для проверки корректности работы классов и миксина.

## Использование:

Запуск основного приложения:
```
python src/main.py
```
Программа выведет:
1. Строковые представления товаров разных типов.
2. Списки товаров по категориям.
3. Результаты корректного сложения товаров одного типа.
4. Демонстрацию обработки ошибок при попытке некорректных операций.
